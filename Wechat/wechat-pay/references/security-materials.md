## 官方规则摘要

- `APIv3概述` 明确：
  - APIv3 使用 JSON 交互
  - APIv3 使用基于非对称密钥的 `SHA256-RSA`
  - 回调中的关键信息使用 `AES-256-GCM` 保护
  - 微信支付公钥模式是推荐验签路径

- `总述-APIv3如何签名和验签` 明确区分：
  - 请求微信支付接口时如何签名
  - 前端调起支付时如何签名
  - 接收微信支付应答时如何验签
  - 接收微信支付回调时如何验签和解密

## 使用建议

- 默认优先使用 `微信支付公钥` 模式回答验签问题，只有用户明确在用平台证书模式时，再切到平台证书文档。
- 如果问题涉及证书迁移、证书轮换、多 `Wechatpay-Serial` 灰度，优先打开：
  - `如何从平台证书切换成微信支付公钥`
  - `平台证书更换操作指引`
- 如果用户问“这些参数到底要准备什么”，优先打开对应模式的 `开发必要参数说明`，不要自己列字段。

## 常见混淆点

- 商户 API 证书不是微信支付公钥，也不是平台证书。
- APIv3 密钥不是签名私钥，主要用于解密。
- 普通商户和服务商的材料集合相似，但参数名不同，必须回到对应模式的参数总表。
