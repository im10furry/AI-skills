# NDA 初稿模板

## 使用方式

- 先判断是单向还是双向；如果双方都会披露资料，优先用双向结构。
- 对主体、目的、期限、争议解决、返还销毁期限等保留占位，不要硬填。
- 如果场景是尽调、PoC、采购、外包、投资或数据室访问，按需补充专门限制。

## 模板

```text
# 保密协议

签署日期: [待确认: YYYY-MM-DD]

本保密协议由以下双方签署:

甲方: [待确认: 主体名称]
乙方: [待确认: 主体名称]

鉴于双方拟就 [待确认: 合作 / 尽调 / PoC / 采购 / 投资] 事宜进行沟通，并可能披露保密信息，双方达成如下约定:

## 1. 定义

1.1 “披露方”指披露保密信息的一方；“接收方”指接收保密信息的一方。
1.2 [单向 NDA 可改为: 披露方为甲方，接收方为乙方]
1.3 “保密信息”是指披露方以书面、口头、电子、视觉、样品、代码、数据、账号、文档或其他形式披露的、标明为保密或依其性质应合理视为保密的信息，包括但不限于技术资料、产品方案、源代码、商业计划、客户信息、价格、测试结果和合作安排。

## 2. 保密信息例外

接收方无需就以下信息承担保密义务:

2.1 在披露时已为公众所知的信息；
2.2 非因接收方违约而成为公众所知的信息；
2.3 接收方在披露前已合法知悉的信息；
2.4 接收方从有权披露的第三方合法获得的信息；
2.5 接收方在未使用披露方保密信息的情况下独立开发的信息。

## 3. 使用限制与保密义务

3.1 接收方仅可为 [待确认: 披露目的] 使用保密信息，不得用于其他目的。
3.2 接收方应采取不低于保护自身同类敏感信息的合理措施保护保密信息，且在任何情况下不得低于合理注意标准。
3.3 未经披露方事先书面同意，接收方不得向任何第三方披露保密信息，但本协议允许的接收人除外。

## 4. 允许接收人

4.1 接收方可仅向其有必要知悉且受不低于本协议保密义务约束的员工、关联公司、律师、会计师、顾问或承包商披露保密信息。
4.2 接收方应对前述人员的违约行为承担责任。
4.3 [待确认: 是否允许向投资人、融资顾问、云服务商或外包商披露]

## 5. 强制披露

5.1 如接收方因法律、法规、法院、监管机构或证券交易所要求必须披露保密信息，接收方应在法律允许的范围内及时通知披露方。
5.2 接收方应在可行范围内协助披露方寻求保护令、保密处理或缩小披露范围。

## 6. 返还、销毁与保留副本

6.1 在披露方要求或双方合作终止后，接收方应在 [待确认: 天数] 内返还或销毁保密信息及其复制件。
6.2 对于因备份、日志、灾备或合规归档而无法立即删除的副本，接收方可继续保留，但该等副本仍受本协议约束。

## 7. 期限

7.1 本协议自签署之日起生效，有效期为 [待确认: 年限]。
7.2 接收方对保密信息的保密义务自披露之日起持续 [待确认: 年限]；属于商业秘密的信息，在其仍构成商业秘密期间持续有效。

## 8. 权利保留

8.1 除为本协议目的而披露保密信息外，披露方未向接收方授予任何知识产权、许可或其他权利。
8.2 双方确认，保密信息的披露不构成任何合作承诺、采购承诺、投资承诺或排他安排。

## 9. 救济

9.1 双方确认，违反本协议可能造成难以弥补的损害，守约方有权寻求禁令救济、继续履行及其他可得救济。
9.2 [待确认: 是否需要责任限制]

## 10. 其他

10.1 本协议构成双方就保密事项的完整协议，并取代此前就同一事项达成的口头或书面约定。
10.2 本协议的修改须经双方书面确认。
10.3 [待确认: 是否允许电子签署]

## 11. 适用法律与争议解决

11.1 本协议适用 [待确认: 法域] 法律。
11.2 因本协议引起的或与本协议有关的争议，提交 [待确认: 管辖法院 / 仲裁机构] 解决。

## 12. 签署

甲方: [待确认]
签署人: [待确认]
职务: [待确认]

乙方: [待确认]
签署人: [待确认]
职务: [待确认]
```

## 双向与单向调整提醒

- 单向 NDA:
  - 明确只有一方是披露方，另一方是接收方。
- 双向 NDA:
  - 保留“披露方 / 接收方”的动态定义，让双方在各自披露时分别受保护。

## 审阅输出建议

- 如果用户提供了对方版本，优先先列“红线 / 可谈判 / 可接受”，再决定是否需要整份重写。
