# 仓库开源发布检查清单

## 最小文件集

- `LICENSE`
  - 放当前项目的主许可证全文或官方标准文本。
- `NOTICE`
  - 当许可证或依赖要求保留通知信息时补充。
- `README.md`
  - 写清许可证名称、SPDX 标识、贡献与发布方式。
- 第三方声明
  - 对第三方代码、字体、图片、模型、数据等单独列出来源与许可证。

## 源码层落地

- 在需要的源码文件里补 SPDX 头。
- 保留原作者版权声明与许可证要求的 notice。
- 不要把第三方许可证文本覆盖成自己的许可证文本。
- 如果是多许可证仓库，按目录或文件粒度说明边界。

## 包与发布元数据

- `package.json`
  - 检查 `license` 字段与仓库实际许可证一致。
- `Cargo.toml`
  - 检查 `license` 或 `license-file`。
- `pyproject.toml` / `setup.cfg`
  - 检查许可证元数据是否一致。
- 容器镜像、安装包、SDK 压缩包
  - 检查是否需要把许可证和 notice 一起分发。

## 发布前最后确认

- 版权归属已核清，贡献代码可被当前主体授权。
- 没有把仅内部使用的密钥、示例数据、商业素材误开源。
- 第三方依赖的许可证义务已记录。
- `README`、包元数据、仓库根目录许可证文本相互一致。
