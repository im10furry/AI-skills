# 数据类别、保留期限与权利清单

## 数据类别盘点

- 账号与身份：用户名、邮箱、手机号、企业名称、实名认证信息
- 使用与设备：`IP`、设备标识、浏览器、操作系统、日志、崩溃信息、访问时间
- 交易与结算：订单、订阅、发票、退款、支付状态
- 内容与互动：上传文件、评论、工单、聊天记录、AI 输入输出
- 营销与分析：Cookie 标识、广告 ID、归因参数、邮件打开/点击数据
- 安全与合规：风控记录、异常登录、封禁记录、申诉材料

## 每类数据都要回答四个问题

1. 为什么处理
2. 保留多久
3. 跟谁共享
4. 用户能否访问、更正、删除、导出、撤回或限制处理

## 审阅时常见缺口

- 只写“我们可能收集信息”，却不列明具体字段或类别
- 只写“用于改进服务”，却不区分核心功能、风控、营销、分析、客服
- 没写保留期限，只写“在必要范围内保留”
- 没写账号注销、删除、导出、申诉渠道
- 没写第三方登录、支付、分析 SDK、云服务商和客服系统
- 真实产品会收集设备标识、日志、定位、通讯录或相册权限，但政策没提

## 输出时的最低标准

- 至少按“我们收集什么 / 为什么 / 和谁共享 / 保留多久 / 你有哪些权利 / 如何联系我们”六段展开
- 如果保留期限暂时无法精确到天或月，写成“按业务场景分层保留”，并保留 `[待确认: ...]`
- 不要承诺做不到的权利时限、删除范围或完全匿名化结果
